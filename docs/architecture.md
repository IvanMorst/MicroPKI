## Component Interactions

1. **CLI вҶ’ CA Module**: User commands trigger certificate operations
2. **CA Module вҶ’ Database**: Persist certificates, track revocation
3. **CA Module вҶ’ Audit Log**: Record all security events with hash chain
4. **Repository Server вҶ’ Database**: Serve certificates via HTTP API
5. **OCSP Responder вҶ’ Database**: Check certificate status in real-time
6. **Client Tools вҶ’ Repository**: Submit CSRs, validate chains, check revocation

## Security Layers

- **Encryption**: Root/Intermediate keys encrypted with AES-256 (PKCS#8)
- **Integrity**: Audit log with SHA-256 hash chain
- **Policy Enforcement**: Key size, validity, SAN restrictions
- **Revocation**: CRL + OCSP with fallback logic
- **Rate Limiting**: Token bucket per client IP
